# Redaction Specification

This document defines all data that must be redacted from diagnostics, logs, exceptions, events, and support exports generated by the Frank Energie integration.

## Goals

- Prevent disclosure of credentials.
- Prevent disclosure of personal information.
- Prevent disclosure of account identifiers.
- Allow diagnostics to remain useful for troubleshooting.
- Comply with Home Assistant diagnostics best practices.

---

## Authentication Data

Always redact:

- Access tokens
- Refresh tokens
- Authorization headers
- Bearer tokens
- Session tokens
- OAuth tokens
- JWT tokens

Examples:

```text
accessToken
refreshToken
access_token
refresh_token
authorization
Authorization
bearer
token
jwt
```

Replacement:

```text
REDACTED
```

---

## URLs

Redact URLs containing credentials or tokens.

Example:

```text
https://api.example.com?token=abc123
```

Becomes:

```text
https://api.example.com?token=REDACTED
```

## Home Assistant Diagnostics

Diagnostics must never contain:

- Passwords
- Tokens
- EAN numbers
- Bank account numbers
- Personal names
- Email addresses
- Phone numbers
- Physical addresses
- Contract productname
- Customers number
- Site Reference

Diagnostics may contain:

- Feature availability
- Entity metadata
- Entity states
- Resolution information
- Coordinator status
- Polling configuration
- Integration version

---

## Logging

Never log:

- Passwords
- Tokens
- Authorization headers
- EANs
- Email addresses

Allowed:

```python
LOGGER.debug("Authentication successful")
LOGGER.debug("Retrieved %s prices", count)
```

Not allowed:

```python
LOGGER.debug("Token: %s", token)
LOGGER.debug("Email: %s", email)
```

---

## Events

Events must never expose:

- Tokens
- EANs
- User identifiers
- Contract identifiers

Events should only contain:

- Action names
- Timing information
- Non-sensitive metadata

---

## Redaction Constant

Recommended implementation:

```python
REDACTED = "REDACTED"
```

All sensitive fields should be replaced with the same constant value for consistency.
